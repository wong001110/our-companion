# Controlled Randomness

## Purpose

Discovery should feel alive, not mechanical.

But random must never bypass trust.

## Discovery Pools

```txt
Core Pool
Gap Pool
Serendipity Pool
Wildcard Pool
```

## Suggested Ratio

For daily proactive discovery:

```txt
70% Core / Gap
20% Serendipity
10% Wildcard
```

## Randomness Rule

Randomness happens after:

- deduplication
- quality filter
- safety filter
- novelty filter
- source quality
- decision timing

## Example

User explores AI Companion.

Core:

```txt
Local-first AI memory
```

Gap:

```txt
Evaluation for AI agents
```

Serendipity:

```txt
Tamagotchi long-term attachment design
```

Wildcard:

```txt
How crows remember faces
```

## Ann Framing

For cross-domain discovery, Ann should explain the connection:

```txt
I know this is not exactly the same topic, but it made me think about long-term companionship.
```
