# Project Evolution Context

## Current Situation

Our Companion is already partially built.

Existing packages may include:

```txt
ai-engine
character-engine
curiosity-engine
database
diary-engine
discovery-engine
insight-engine
journey-engine
memory-engine
pattern-engine
speech-engine
tool-engine
shared
```

This means the project is not starting from zero.

## Target Situation

The target is not simply to add features.
The target is to evolve the project into a long-term Companion system with:

- local-first cognitive memory
- internet discovery
- future cloud sync
- future Companion visit system
- character animation and action performance
- creator system for custom personalities and assets

## Evolution Principle

The correct path is:

```txt
Existing Project
  ↓
Compatibility Layer
  ↓
Event-driven Foundation
  ↓
Logical Architecture Alignment
  ↓
Selective Refactor
  ↓
Future Expansion
```

Never jump directly from existing project to final architecture in one step.

## Migration Attitude

Argent should treat the current project as valuable.

Existing behavior is proof of product direction.
Refactor should preserve what works and create room for what comes next.
