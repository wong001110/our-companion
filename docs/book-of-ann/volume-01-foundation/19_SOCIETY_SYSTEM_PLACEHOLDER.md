# Society System Placeholder

The future Companion Society is not implemented in Volume 01.

But the architecture must not block it.

## Future Features

- cloud sync
- Companion visits
- Companion-to-Companion conversation
- relationship memory
- affinity and personality compatibility
- knowledge exchange
- shared journey
- privacy boundaries

## Foundation Requirement

All external or future Companion input must enter as Signal or Experience.

Example:

```txt
Karl shared a concept with Ann
  ↓
SignalCaptured(sourceType='companion')
  ↓
Discovery / Concept / Knowledge pipeline
```

## Trust Boundary

Future shared knowledge must respect privacy.

Every memory and experience should eventually support:

```txt
private
local
shareable
public
```

## Volume 01 Rule

Do not build networking now.
Only reserve interfaces and source types.
