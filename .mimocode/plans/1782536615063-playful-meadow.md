# Volume 7 — Companion Validation Kit (CVK) Implementation Plan

## Overview

Volume 7 defines the **validation framework** for Our Companion — simulation, inspection, event triggering, data management, runtime control, debug overlay, and test suites. It introduces no new Companion behavior; it ensures existing behavior is observable, reproducible, and testable.

## Build Phases (per 13-BUILD.md)

### Phase 1: Foundation + Simulation + Scenarios
**Goal**: Developers can simulate long-term Companion behavior.

**New package**: `packages/validation-kit/`

**Files to create**:
1. `packages/validation-kit/package.json`
2. `packages/validation-kit/tsconfig.json`
3. `packages/validation-kit/src/index.ts`
4. `packages/validation-kit/src/simulation/types.ts` — SimulationCategory, TimeAdvance, SimulationConfig
5. `packages/validation-kit/src/simulation/simulation-engine.ts` — time advance, context override, runtime force, relationship/journey/discovery/memory/notebook simulation
6. `packages/validation-kit/src/simulation/time-simulator.ts` — +1min/+10min/+1hr/+1day/+7days/+30days, custom date
7. `packages/validation-kit/src/scenarios/types.ts` — Scenario, ScenarioComponent, ScenarioCategory
8. `packages/validation-kit/src/scenarios/scenario-loader.ts` — load/save/list scenarios
9. `packages/validation-kit/src/scenarios/built-in-scenarios.ts` — Fresh, Day1, OneWeek, OneMonth, LongTerm, ReturnAfterAbsence, HeavyDiscovery, JourneyCompletion
10. `packages/validation-kit/src/triggers/types.ts` — TriggerCategory, TriggerConfig, TriggerResult
11. `packages/validation-kit/src/triggers/event-trigger.ts` — fire events through event bus
12. `packages/validation-kit/src/triggers/macro-engine.ts` — reusable event sequences
13. `packages/validation-kit/src/triggers/trigger-history.ts` — record and replay triggers

**Key behaviors**:
- Simulation modifies the world (time, context, state) not business logic
- Scenarios restore complete Companion worlds with one action
- Event triggers fire through the normal event bus (no bypassing)
- All operations are reversible and isolated
