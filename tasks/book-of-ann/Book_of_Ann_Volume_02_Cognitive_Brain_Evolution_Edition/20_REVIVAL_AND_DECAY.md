# Revival and Decay

## Decay

Old discoveries should not remain equally active forever.

Decay applies to:

- ignored discoveries
- inactive concepts
- completed journeys
- old curiosity gaps
- stale source items

## Decay States

```txt
active
cooling
dormant
archived
```

## Revival

A dormant concept can revive when:

- there is a meaningful update
- multiple sources mention it again
- user returns to related journey
- another Companion shares related knowledge in future

## Revival Decision

Revival should not look like duplicate recommendation.

Ann should frame it as continuity:

```txt
Remember that local-first memory direction? I found a new development.
```

## Events

```txt
ConceptDormant
DiscoveryArchived

ConceptRevived
```
