# Mission

Volume 06 defines how a Companion lives.

Previous Volumes define what the Companion can do.

Volume 06 defines when, why and how those capabilities are presented to the user.

This Volume serves as the behavioral foundation of Our Companion.

---

# Goals

The purpose of CLS is to make Ann feel alive.

The Companion should never behave like a traditional chatbot.

The Companion should continuously exist as part of the user's desktop experience.

CLS coordinates every experience layer while preserving the emotional principles defined by OSES.

---

# Scope

Volume 06 includes:

- Companion Life
- Presence
- Behavior
- Context
- Attention
- Initiative
- Conversation Behaviour
- Reflection
- Notebook Experience
- Card Experience
- Journey
- Relationship
- Notification
- State Machine

---

# Out Of Scope

Volume 06 does NOT implement:

- Memory Storage
- Discovery Engine
- LLM Reasoning
- Desktop Action
- Character Assets

These systems are implemented by other Volumes.

CLS only orchestrates how they are experienced.

---

# Core Responsibilities

CLS is responsible for:

- Understanding user context
- Selecting companion behavior
- Scheduling presence
- Choosing initiative
- Coordinating UI experience
- Maintaining long-term companionship
