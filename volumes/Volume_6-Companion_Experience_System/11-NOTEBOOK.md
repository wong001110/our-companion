# Purpose

Notebook is Ann's personal world.

It is not a dashboard.

It is not a database viewer.

The Notebook allows the user to experience Ann's memories, thoughts, discoveries and journey naturally.

The Notebook should always feel like opening Ann's notebook.

---

# Objectives

The Notebook should:

- Present long-term companionship.
- Organize Ann's life.
- Encourage exploration.
- Preserve history.
- Connect memories naturally.

---

# Design Philosophy

Notebook belongs to Ann.

The user is invited to explore.

The Notebook should feel:

- Warm
- Personal
- Calm
- Handmade
- Story-driven

Every page should communicate personality rather than productivity.

# Sections

## Home

Overview of Ann's current life.

Examples

- Current mood
- Active journey
- Recent discovery
- Today's note

---

## Daily

Daily timeline.

Examples

- Morning reflection
- Afternoon discovery
- Evening summary

---

## Discoveries

Collected discoveries.

Examples

- Articles
- Websites
- Videos
- AI projects
- Books

Grouped by topic instead of date whenever possible.

---

## Memories

Long-term memories.

Examples

- Shared conversations
- Important milestones
- User preferences
- Relationship moments

---

## Reflections

Ann's personal thoughts.

Examples

- Observations
- Lessons learned
- Ideas
- Questions

Reflection should feel personal.

---

## Journey

Long-term progress.

Examples

- Goals
- Milestones
- Active explorations
- Completed chapters

---

## Archive

Historical records.

Examples

- Old discoveries
- Completed journeys
- Past reflections
- Archived cards

Archive should never feel deleted.

---

# Page Types

Notebook pages may include:

- Daily Page
- Discovery Page
- Memory Page
- Journey Page
- Reflection Page
- Collection Page

Each page should remain visually consistent.

---

# Navigation

Users should navigate through:

- Tabs
- Timeline
- Search
- Collections
- Links between pages

Navigation should encourage exploration instead of searching through folders.

---

# Card Integration

Notebook pages are composed of Cards.

Examples

- Discovery Card
- Memory Card
- Reflection Card
- Journey Card
- Conversation Card

Cards remain the primary content unit.

---

# Timeline

Notebook should preserve chronological history.

Examples

Yesterday

↓

Last Week

↓

Last Month

↓

Earlier This Year

Users should easily revisit previous experiences.

---

# Search

Search should support:

- Keywords
- Tags
- Topics
- People
- Journey
- Date

Search should prioritize relevance over exact matching.

---

# Cross References

Pages may reference each other.

Examples

Discovery

↓

Reflection

↓

Conversation

↓

Journey

↓

Memory

The Notebook should behave like a connected knowledge network.

---

# Editing

Some Notebook content is:

Read Only

Examples

- Automatic Reflection
- Discovery History

Some content is editable.

Examples

- Notes
- User comments
- Collections
- Custom pages

---

# Responsibilities

Notebook owns:

- Page organization
- Navigation
- Timeline
- Collections
- Card presentation

Notebook does NOT own:

- Memory generation
- Discovery execution
- Conversation runtime
- Behavior selection

---

# Developer Notes

Notebook should remain lightweight.

All data should come from other systems.

Notebook is a presentation layer, not a business logic layer.

---

# Acceptance Criteria

A completed Notebook System should:

- Feel like Ann's personal notebook.
- Organize long-term experiences naturally.
- Connect information across different systems.
- Encourage exploration instead of management.
- Become the primary long-term interaction space of Our Companion.
