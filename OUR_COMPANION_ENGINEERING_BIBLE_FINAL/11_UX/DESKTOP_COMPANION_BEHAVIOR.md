# Desktop Companion Behavior

## Interaction
- Character wanders on desktop.
- Click opens Companion Panel.
- Character can be dragged.
- Dragging may trigger small character reaction.
- Mouse should mostly pass through except character hitbox.

## Visual performance
Task execution should use visual acting:
- run toward Chrome
- tap/gesture
- actual tool executes directly
- return and report result

## Do not simulate real clicking for simple actions.
Use OS commands/deep links for reliability.
